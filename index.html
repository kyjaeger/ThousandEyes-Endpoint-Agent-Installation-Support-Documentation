ThousandEyes Endpoint Agent Deployment Guide

A step-by-step workflow for getting Endpoint Agents installed on end-user machines and monitoring user experience from day one.

The ThousandEyes Endpoint Agent is a lightweight application installed on end-user Windows or macOS machines. It provides visibility into the complete network path from the user's device — through Wi-Fi, corporate LAN, VPN, and ISP — all the way to cloud applications like Microsoft 365, Webex, Zoom, and Salesforce.

What Endpoint Agents Measure

Unlike Enterprise Agents that monitor from fixed network points, Endpoint Agents follow your users. They capture:

  • Scheduled tests — continuous synthetic monitoring of target applications
  • Browser sessions — automatic capture of real user experience data in Chrome or Edge
  • Local network metrics — Wi-Fi signal quality, gateway latency, DNS performance, VPN status
  • Network topology — full path from device to destination with per-hop visibility

🔎
Proxy Environments
If end users route traffic through a proxy, the Endpoint Agent must be configured to use the same proxy. You can pre-configure proxy settings in the installer or push them via GPO/MDM after installation. See the proxy support documentation for details.
Phase 1

Download & Install the Endpoint Agent

Obtain the installer from the ThousandEyes portal and deploy it to end-user machines.

Step 1: Get the Installer from the Portal

1

Log in to app.thousandeyes.com with your admin credentials.

2

Navigate to Endpoint Agents > Agent Settings in the left navigation menu.

3

Click Add New Endpoint Agent. Select your target OS (Windows or macOS) and download the installer package.

4

On the same screen, locate and copy the connection string — you will need this during agent setup in Step 2 to associate the agent with your ThousandEyes organization.

Step 2: Install on End-User Machines

Run the downloaded .msi installer as a local administrator.

1

Right-click the .msi file and select Run as administrator.

2

Follow the installation wizard. Accept the license agreement and click Install.

3

When prompted, install the ThousandEyes Recorder extension for Chrome or Edge to enable browser session capture (highly recommended).

4

When prompted during setup, enter the connection string copied from the portal in Step 1. This associates the agent with your ThousandEyes organization. The agent service will start automatically once registration is complete, and a ThousandEyes icon will appear in the system tray.

💡
Silent Install for IT Deployment
For silent/unattended installation via SCCM, Intune, or similar tools, use: msiexec /i ThousandEyesEndpointAgent.msi /quiet /norestart

Run the downloaded .pkg installer on macOS.

1

Double-click the .pkg file to launch the installer.

2

Follow the prompts. You will be asked to enter your macOS admin password to authorize the installation.

3

When prompted during setup, enter the connection string copied from the portal in Step 1. This associates the agent with your ThousandEyes organization.

4

macOS may prompt you to allow the ThousandEyes system extension in System Settings > Privacy & Security. Click Allow.

5

Install the ThousandEyes Endpoint Agent Extension in Chrome or Edge for browser session capture.

⚠
macOS System Extension Approval
On macOS 13+, the system extension must be explicitly approved by the user or pre-approved via MDM (e.g., Jamf). Without approval, network path data will be limited. Pre-approving via MDM is strongly recommended for enterprise rollouts.

For large-scale deployments, distribute via your existing endpoint management tooling.

Windows — Group Policy / Intune / SCCM

Use the .msi package with silent install flags. Deploy as a required application in your management console targeting the appropriate device groups. The browser extension can be force-installed via Chrome or Edge policy.

macOS — Jamf / MDM

Upload the .pkg to Jamf Pro (or your MDM). Create a policy targeting the device scope and deploy as a required package. Use a configuration profile to pre-approve the system extension (com.thousandeyes.endpoint-agent) and network extension to avoid user prompts.

💡
Tip
For initial rollouts, target a single team or site (e.g., 20–50 machines) before deploying broadly. This lets you validate connectivity and confirm agents appear in the portal before scaling. Coordinate with your IT/endpoint team early — MDM pre-approval for macOS is the most common deployment blocker.
Phase 2

Verify Agent Check-In

Confirm agents are registering successfully in the ThousandEyes portal.

Confirm Agents Appear in the Portal

1

Navigate to Endpoint Agents > Agent Settings in the ThousandEyes portal.

2

Within a few minutes of installation, the device should appear in the agent list with status Online (green indicator).

3

Click any agent to inspect its details: OS version, IP address, last check-in time, installed agent version, and browser extension status.

What to Check Per Agent

🔎
Troubleshooting — Agent Not Appearing
If an agent doesn't show up within 5 minutes: (1) confirm the machine has outbound TCP 443 access, (2) verify the correct installer was used (associated with your ThousandEyes org), (3) check the agent log at C:\ProgramData\ThousandEyes\Endpoint Agent\log (Windows) or /Library/Application Support/ThousandEyes/Endpoint Agent/log (macOS).
Phase 3

Organize Agents into Groups

Use agent labels to segment users by location, department, or network type for targeted monitoring.

Agent labels allow you to group Endpoint Agents and assign tests to logical segments — for example, all agents at a branch office, all remote users on VPN, or all agents in a specific business unit.

Creating and Applying Agent Labels

1

In the ThousandEyes portal, go to Endpoint Agents > Agent Settings.

2

Select one or more agents using the checkboxes, then click Labels > Add Label.

3

Create a meaningful label name such as Branch-Austin, Remote-VPN, or Finance-Team.

4

When creating tests in Phase 4, you will assign them to these labels rather than individual agents — making it easy to scale coverage as more devices are enrolled.

Recommended Label Strategy

LabelRationale
By site/location (e.g., Site-NYC, Site-Chicago)Compare user experience across office locations
By network type (e.g., OnPrem, VPN, DirectInternet)Isolate whether issues are VPN-related vs. general
By department (e.g., Sales, Engineering)Prioritize monitoring for high-value business units
Pilot group (e.g., Pilot-Wave1)Separate initial deployment agents for validation
💡
Tip
Even for initial pilots, take the 5 minutes to set up labels. It pays off immediately when you create tests — assigning to Pilot-Wave1 is far easier than selecting 20 individual agents. It also makes scaling to a broader rollout much more straightforward.
Phase 4

Create Endpoint Tests

Configure scheduled tests to continuously monitor the applications your users depend on.

Navigate to Endpoint Agents > Test Settings and click Add New Test. Assign each test to an agent label created in Phase 3.

Test 1: HTTP Server (Application Reachability)

Use Case

Monitor whether end users can successfully reach critical web applications — Microsoft 365, Salesforce, Zoom, Webex, internal apps, etc. — from their devices and measure response time from the user's network vantage point.

What You Get

  • Availability percentage and HTTP response codes per agent
  • Response time breakdown: DNS, TCP connect, SSL/TLS, TTFB
  • Path Visualization from each user device to the target — showing Wi-Fi, gateway, ISP, and cloud hops
  • Side-by-side comparison between office users, VPN users, and remote users

Test 2: Agent-to-Server (Network Path Monitoring)

Use Case

  • Packet loss, latency, and jitter from every enrolled device
  • Path Visualization with hop-by-hop detail, ASN data, and MPLS labels
  • Trend data to distinguish transient vs. sustained path degradation

Test 3: Browser Session Capture (Real User Monitoring)

Use Case

Beyond scheduled tests, the Endpoint Agent can passively capture real browser sessions when users visit monitored domains. This provides actual end-user experience data without any user action required.

Configuration

1

Navigate to Endpoint Agents > Browser Sessions > Monitored Domain Sets.

2

Click Add New Monitored Domain Set and enter the domains to monitor (e.g., *.microsoft.com, zoom.us, salesforce.com).

3

Assign the domain set to the relevant agent labels. The browser extension handles capture automatically once configured.

What You Get

  • Real page load timings from actual user sessions — not synthetic probes
  • Waterfall data showing load time per page resource
  • Network path captured at the moment of each user visit
  • Local network context — Wi-Fi SSID, signal strength, gateway latency at time of session
💡
Browser Extension Required
Browser session capture requires the ThousandEyes Endpoint Agent browser extension installed in Chrome or Edge. Confirm extension status for all agents under Agent Settings before enabling monitored domains.
💡
Recommended Starter Test Package
A good starting point is: one HTTP Server test to your most critical cloud app (e.g., M365 or Zoom), one Agent-to-Server test to your VPN gateway (if applicable), and a monitored domain set covering 3–5 key SaaS domains. This provides immediate, meaningful visibility across your most important services.
Phase 5

Validate & Next Steps

Confirm the deployment is working end-to-end and identify opportunities to expand monitoring coverage.

Validation Checklist

  • Endpoint Agents appear in Endpoint Agents > Agent Settings with "Online" status
  • Browser extension shows as "Installed" on all agents
  • Scheduled tests (HTTP Server, Agent-to-Server) return data within 10 minutes of creation
  • Path Visualization displays hop-by-hop path data for network tests
  • Browser Sessions view shows captured sessions for monitored domains
  • Alert rules are active and configured with appropriate thresholds
  • Agents are correctly labeled and tests are assigned to labels (not individual agents)

Recommended Next Steps

  • Explore the Endpoint Agents > Views dashboards — Network Topology, Browser Sessions, and Scheduled Tests
  • Configure alert notifications (email, Slack, PagerDuty, webhook) so the right teams are notified on degradation
  • Expand the monitored domain set to cover additional SaaS applications
  • Roll out agents to additional user groups beyond the initial pilot
  • Explore integrating ThousandEyes alerts into your ITSM platform (ServiceNow, etc.)
  • Schedule a review 2 weeks post-deployment to assess findings and adjust test coverage

Licensing Reference

Endpoint Agent licensing is per-seat — each enrolled device consumes one license. There are three license tiers, each offering a different level of monitoring capability.

Advantage

The most comprehensive Endpoint Agent license and the default assigned to all new agents. Advantage supports up to 10 concurrent tests per agent (across dynamic and scheduled), real-user browser session capture, full path visualization, and always-on local network metrics, with 30-day data retention. This is the recommended license for users whose experience with critical applications needs to be actively monitored and troubleshot end-to-end.

Essentials

A mid-tier license that supports up to 4 concurrent tests per agent and 14-day data retention. Essentials includes scheduled synthetic tests, path visualization, and local network metrics, but does not include browser session capture. Agents are assigned an Essentials license when Advantage licenses are not available in the org's pool — making license availability management an important operational consideration at scale.

Embedded

A lightweight entitlement available exclusively through specific Cisco product bundles (e.g., Cisco Secure Access) — it is not sold standalone. Embedded is assigned automatically at purchase and supports 1 dynamic test and 1 scheduled test per agent, with 4-day data retention. It provides baseline local network health metrics but does not include browser session capture or path visualization. If your organization holds qualifying Cisco subscriptions, check whether Embedded entitlements are already available before purchasing additional licenses.

💡
Tip
Advantage is the default license for all new agents and provides the most complete visibility — including browser session data and 30-day retention. If your organization also holds qualifying Cisco product subscriptions (e.g., Cisco Secure Access), check whether Embedded entitlements are already included before purchasing additional licenses.
